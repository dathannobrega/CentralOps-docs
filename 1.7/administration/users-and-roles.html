Usuários e Papéis

Criar contas, atribuir papéis, resetar senhas, desativar usuários.

Público: Admin only.

Acessar administração de usuários​

Vá a Administração → Usuários (canto superior direito, menu de admin).

Interface​

Lista de usuários​

Tabela com:

ColunaDescrição
EmailEndereço de login.
PapelViewer, Operator, Engineer, Admin.
OrganizaçãoQual org o usuário pertence.
StatusAtivo, desativado.
Último loginTimestamp ou "nunca".
AçõesEditar, resetar senha, desativar.

Criar novo usuário​

Passo 1: Abrir modal "Novo usuário"​

Clique + Novo usuário (canto superior direito).

Passo 2: Preencher formulário​

Email:          nome@empresa.com
Papel:          Operator (dropdown)
Organização:    Default ou selecione (admin só)
Senha inicial:  (gerar automaticamente)

Geração de senha:

  • Sistema gera senha aleatória de 16 caracteres.
  • Exibida 1 vez só — anote ou copie.

Passo 3: Salvar e compartilhar​

  1. Clique Criar.
  2. Sistema retorna link de reset de senha.
  3. Copie link.
  4. Compartilhe com novo usuário (email, Slack, etc).
  5. Novo usuário clica link → define sua própria senha.

Alternativa: SMTP ativado​

Se você configurou SMTP (Plataforma Config):

  1. Sistema envia e-mail automático ao novo usuário com link de reset.
  2. Você não precisa compartilhar manualmente.

Editar papel de usuário​

Passo 1: Localize usuário na lista​

Busque por email (campo de busca no topo).

Passo 2: Clique em ações​

Clique ações → Editar papel.

Passo 3: Selecione novo papel​

Dropdown com 4 opções:

  • Viewer
  • Operator
  • Engineer
  • Admin

Clique Salvar.

Efeito imediato:

  • Permissões mudam na próxima ação do usuário.
  • Sessão ativa continua com papel antigo (para não interromper).

Resetar senha​

Cenário: usuário esqueceu senha​

  1. Clique ações → Reset de senha.
  2. Link de reset gerado.
  3. Copie link → compartilhe com usuário.
  4. Usuário clica → define nova senha.

Link expira em 1 hora. Se expirar, gere novo.

Desativar usuário​

Cenário: empregado saiu da empresa​

  1. Clique ações → Desativar.
  2. Confirmação: "Tem certeza? Usuário não poderá mais fazer login."
  3. Clique Desativar.

Efeito:

  • Sessões ativas são revogadas imediatamente.
  • Usuário não consegue fazer login.
  • Histórico de auditoria fica intacto.

Reativar​

Se funcionário volta:

  1. Clique ações → Ativar.
  2. Usuário pode fazer login novamente.

Permissões por papel​

Leia RBAC para matriz completa.

Resumo:

PapelResumo
ViewerLê tudo, não executa ações.
OperatorTriage (discard/reprocess), pausa integrações.
EngineerEdita mappings, marca drift, backfill.
AdminCria usuários, gerencia orgs, secrets, compliance.

Boas práticas​

Princípio do menor privilégio​

  • Novo funcionário começa como Viewer ou Operator.
  • Promove para Engineer após prova de competência.
  • Admin apenas para responsáveis de infraestrutura/compliance.

Aprovisionamento/desaprovisionamento​

  • Onboard: crie conta 1º dia, compartilhe link reset.
  • Offboard: desative conta assim que saída é confirmada.
  • Auditoria: verifique periodicamente (ex: 90 dias) se contas inativas podem ser deletadas.

Senhas​

  • CentralOps não impõe política de expiração (roadmap: LDAP/SSO).
  • Reset ocasional via interface é suficiente.
  • Senhas são hashed bcrypt, nunca armazenadas em plain text.

Limitações​

  • Sem grupos: hoje, papéis são por usuário. Grupos são roadmap.
  • Sem LDAP/AD: integração com Active Directory é roadmap.
  • Sem delegação: Admin não pode delegar gerenciamento de usuários a outro Admin.

  • Criar org separada para cliente? Vá a Organizações.
  • Gerenciar credenciais de integrações? Vá a Integrações.
  • Auditoria de quem acessou o quê? Vá a Auditoria.
