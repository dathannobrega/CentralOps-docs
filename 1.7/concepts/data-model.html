Modelo de dados

CentralOps centraliza entidades de segurança em um modelo relacional unificado.

Entidades principais​

Organization​

Propósito: isolamento de dados multi-tenant.

Organization (slug, name, settings)
  ├─ users (N users)
  ├─ integrations (N integrations)
  ├─ mappings (N mapping versions)
  └─ retention_config (retenção por tipo de dado)

Cada org tem dados completamente isolados. Um Engineer de org A não vê dados de org B, mesmo na mesma instância.

Default: CentralOps cria 1 org ao bootstrap. Para MSSP, crie 1 org por cliente.

Integration​

Propósito: credenciais cifradas + estado de coleta de um vendor.

Integration (id, platform, name, state)
  ├─ metadata (JSON criptografado: client_id, client_secret, etc)
  ├─ health (último status, timestamp)
  ├─ collection_state (cursor, last_run)
  └─ organization_id (FK → Organization)

Campos importantes:

  • platform: "sophos", "wazuh", "microsoft_defender", etc.
  • state: "active", "paused", "error", "awaiting_first_collection".
  • metadata: dict serializado + cifrado com APP_MASTER_KEY.

MappingDefinition​

Propósito: agrupa versões de um mapping DSL.

MappingDefinition (id, platform, stream)
  ├─ versions (N MappingVersion)
  └─ organization_id (FK)

Exemplo: "sophos.alerts", "wazuh.security", "ninjaone.activities".

Cada definição tem N versões (v1, v2, v3, ...). Uma é current, outras são histórico para rollback.

MappingVersion​

Propósito: conteúdo DSL de um mapping específico.

MappingVersion (id, mapping_def_id, version_number, dsl_content)
  ├─ created_at
  ├─ created_by (user_id)
  └─ is_current (boolean)

Quando você edita um mapping:

  1. Nova versão criada.
  2. Marcada como is_current = true.
  3. Versão anterior fica em histórico.
  4. Você pode revert para versão anterior com 1 clique.

Todas as versões são imutáveis (append-only).

RawEvent​

Propósito: evento conforme recebido da API de vendor.

RawEvent (id, integration_id, vendor_event_id)
  ├─ payload (JSON bruto)
  ├─ received_at
  ├─ processing_state (success, failed, pending)
  └─ error_details (se failed)

Normalmente não visível ao usuário, exceto em quarentena quando processing_state = failed.

NormalizedEvent​

Propósito: evento após transformação por mapping.

NormalizedEvent (id, raw_event_id, integration_id)
  ├─ normalized_data (JSON canônico)
  ├─ severity
  ├─ timestamp
  ├─ title
  └─ sent_to_wazuh_at

Apenas eventos com processing_state = success em RawEvent geram NormalizedEvent.

QuarantineEvent​

Propósito: eventos que falharam validação ou transformação.

QuarantineEvent (id, integration_id, raw_event_id)
  ├─ raw_payload
  ├─ error_message (ex: "Required field 'timestamp' missing")
  ├─ error_type (validation_error, type_error, parsing_error)
  ├─ created_at
  ├─ status (pending, discarded, reprocessed)
  └─ organization_id (FK)

Ações do usuário:

  • discard: marca como ignorado, não reprocessa.
  • reprocess: move de volta para fila após mapping fix.

Retenção: configurável por org, default 7 dias.

UnknownField (Drift)​

Propósito: campos em RawEvent que não estão no mapping DSL.

UnknownField (id, vendor, field_name, field_type)
  ├─ count (quantas vezes visto)
  ├─ last_seen
  ├─ status (unknown, ignored, marked_mapped)
  └─ organization_id (FK)

Exemplo: Sophos retorna novo campo ai_score_numeric. Vai para drift automaticamente.

Ações do usuário:

  • ignore: não alerta mais sobre esse campo.
  • mark_mapped: indica que já está tratado (ex: via custom field no mapping).

AuditLog​

Propósito: rastreabilidade de todas as ações do usuário.

AuditLog (id, user_id, action, resource_type, resource_id)
  ├─ action (create_integration, edit_mapping, discard_quarantine, etc)
  ├─ timestamp
  ├─ details (JSON: o que foi alterado)
  ├─ ip_address
  └─ organization_id (FK)

Imutável — nunca deletado antes de AUDIT_LOG_RETENTION_DAYS (default 365 dias).

Inclui:

  • Quem fez.
  • O quê foi feito.
  • Quando.
  • Em qual organização.
  • Mudanças específicas (ex: "severity map changed from 1→5 to 1→critical").

UserSession​

Propósito: sessões autenticadas.

UserSession (id, user_id, token_hash)
  ├─ expires_at
  ├─ user_agent
  └─ created_at

Cada login cria nova sessão. Token é hash SHA256 (nunca armazenado em plain text).

Sessão expira após SESSION_TTL_HOURS (default 24h).

Relacionamentos​

Organization
  ├─ 1:N → AppUser (users)
  ├─ 1:N → Integration (integrations)
  ├─ 1:N → MappingDefinition (mappings)
  ├─ 1:N → AuditLog (audit logs)
  └─ 1:1 → RetentionConfig (policies)

Integration
  ├─ N:1 → Organization
  ├─ 1:N → RawEvent
  └─ 1:N → QuarantineEvent

MappingDefinition
  ├─ N:1 → Organization
  ├─ 1:N → MappingVersion
  └─ (platform, stream) é único por org

RawEvent
  ├─ N:1 → Integration
  └─ 1:1 → NormalizedEvent (se success)

AppUser
  ├─ N:1 → Organization
  └─ 1:N → UserSession

AuditLog
  ├─ N:1 → AppUser (pode ser NULL para eventos de sistema)
  └─ N:1 → Organization

Criptografia​

Seguintes campos são AES-256-GCM criptografados em repouso:

  • Integration.metadata (client_id, client_secret, etc).
  • Integration.access_token.
  • Integration.refresh_token.
  • AppUser.password_hash (hashed, não encrypted).
  • AuditLog.details (opcional, por compliance).

Chave de criptografia é derivada de APP_MASTER_KEY via KDF (PBKDF2).

│  Organization                                                   │
│  ├─ id (PK)                                                     │
│  ├─ slug (unique)                                               │
│  └─ retention_config                                            │
└─────────────────────────────────────────────────────────────────┘
         │          │            │                │
         │ 1:N      │ 1:N        │ 1:N            │ 1:N
         ↓          ↓            ↓                ↓

     AppUser    Integration  MappingDef      AuditLog
        │            │           │              │
        └─ Role      ├─ State    └─ 1:N vers   └─ user_id (FK)
        └─ Email     ├─ Health       │           └─ action
        └─ Org_id    ├─ Metadata     └─ v1      └─ timestamp
                     └─ Org_id           v2
                                         v3 (current)
                                         v4 (history)
         │
         1:N
         ↓
     UserSession
        ├─ token_hash
        └─ expires_at

      Integration 1:N → RawEvent ← 1:1 → NormalizedEvent
                   │       ↓
                   └→ QuarantineEvent (status = failed)

      Vendor → UnknownField (drift)
                   ├─ vendor
                   ├─ field_name
                   └─ status
