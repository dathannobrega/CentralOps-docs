Sophos Central Integration

Conecte Sophos Central para coletar alertas e cases em tempo real.

Público: Admin (criar integração), Viewer+ (ver dados).

Pré-requisitos​

  • ✓ Você é admin em Sophos Central (sua organização).
  • ✓ Acesso a Settings → API Credentials (gerar Client ID/Secret).
  • ✓ Você tem acesso admin em CentralOps.

Passo 1: Gerar credenciais OAuth em Sophos​

1.1 Abrir Sophos Central​

Vá a https://central.sophos.com (ou regional: eu, apac, etc).

1.2 Settings → API Credentials​

Navbar → Settings → API Credentials (ou Settings → Organization → API Keys).

1.3 Criar novo credential​

Clique Create (ou Add API Credential).

Nome:        CentralOps (ou algo descritivo)
Descrição:   Para coleta de alertas em CentralOps

1.4 Selecionar permissões​

Marque no mínimo:

  • alerts:read — necessário para coletar alertas.
  • cases:read — se quer coletar cases (XDR/Managed Threat Response).

Deixe outras desmarcadas (princípio do menor privilégio).

1.5 Gerar e anotar​

Clique Create → sistema exibe:

Client ID:      abc123xyz...
Client Secret:  secretxyz... (exibido 1 vez, copie já)

Salve em local seguro (Notes, Vault, etc). Não será exibido novamente.

Passo 2: Identificar sua região e tenant​

2.1 Região​

Veja URL do Sophos:

  • https://api-us.central.sophos.com → US
  • https://api-eu.central.sophos.com → EU
  • https://api-apac.central.sophos.com → APAC
  • https://api-au.central.sophos.com → AU (Australia)

Anote sua região.

2.2 Tenant ID​

Vá a Settings → Organization.

Procure por campo Organization ID (longo hash tipo 00c123abc...).

Anote Tenant ID.

Passo 3: Criar integração em CentralOps​

3.1 Abrir CentralOps​

Vá a Administração → Integrações.

Clique + Adicionar integração.

3.2 Selecionar plataforma​

Dropdown → Sophos Central.

Clique Próximo.

3.3 Preencher credenciais​

Nome:              Sophos - Acme Corp (identificação)
Client ID:         abc123xyz... (do passo 1.5)
Client Secret:     secretxyz... (do passo 1.5)
Tenant ID:         00c123abc... (do passo 2.2)
Region:            EU (ou sua região)
Organization:      (deixe em branco se é a padrão)

3.4 Testar conexão​

Clique Testar conexão.

Sistema valida:

  • Client ID/Secret válidos.
  • Tenant ID existe.
  • Credenciais têm permissão alerts:read.

Se tudo OK: "✓ Conexão bem-sucedida".

Se erro: veja mensagem. Causas comuns:

  • Client ID/Secret errados (copie novamente).
  • Tenant ID errado.
  • Permissões insuficientes (marque alerts:read em Sophos).
  • Region errada.

3.5 Salvar​

Clique Salvar.

Sistema:

  • Criptografa credenciais.
  • Armazena em banco.
  • Agenda primeira coleta em próximo 1-2 min (via RedBeat).

Status inicial: "Aguardando primeira coleta".

Passo 4: Aguardar primeira coleta​

Aguarde 2-3 minutos. Celery Beat executa collector.

Se Sophos tem alertas recentes:

  • Status muda para "Ativo".
  • Health vira verde.
  • Eventos aparecem em Operação → Alertas (ou Quarentena se mapping não existe).

Se nada apareceu:

  • Verifique se Sophos tem alertas (logue em Sophos, abra Alerts).
  • Veja status integração: há erro?
  • Verifique logs: docker compose logs collector-worker-priority | grep sophos.

Passo 5: (Opcional) Configurar coleta de Cases​

Se você quer XDR cases (Managed Threat Response), adicione permissão:

  1. Volte a Sophos → Settings → API Credentials.
  2. Clique no credential → Edit Permissions.
  3. Marque cases:read.
  4. Salve.
  5. Volte a CentralOps → integração Sophos.
  6. Clique Testar conexão (valida nova permissão).

Sistema automaticamente coleta cases também (se XDR ativado em Sophos).

Troubleshooting​

"401 Unauthorized"​

Causa: Client ID/Secret inválidos ou expirados
Ação:
  1. Gere novo credential em Sophos (delete o antigo)
  2. Copie novo Client ID/Secret
  3. Volte a CentralOps → edite integração
  4. Preencha novos valores
  5. Teste conexão novamente

"Nenhum evento aparecendo"​

Causa: Sophos não tem alertas recentes OU mapping não existe
Ação:
  1. Abra console Sophos → Alerts
  2. Há alertas lá? Se não, crie teste (ex: trigger detecção de malware falso)
  3. Espere 5-10 min
  4. Se ainda nada: verifique quarentena (mapping error?)

"Taxa de eventos muito lenta"​

Causa: Sophos API está limitando (429 Too Many Requests)
Ação:
  1. Vá a **Pipeline Health** → integração Sophos
  2. Vê "Rate limited (429)"?
  3. Aumentar intervalo de coleta:
     - Padrão: 1 min
     - Aumentar para: 5 min
  4. Contatar suporte Sophos sobre limite de rate

"Integração parada, erro de token"​

Causa: Token expirou, refresh falhou
Ação:
  1. Integração tenta renovar token automaticamente
  2. Se falha: reset credential (gere novo em Sophos)
  3. Teste conexão

Streams disponíveis​

StreamDescriçãoFrecuênciaNecessário?
alertsDetecções de malware, comportamento, etc.1 minSim
casesXDR cases (Managed Threat Response).3 minOpcional

Dados coletados (exemplo)​

Cada alerta Sophos contém:

{
  "id": "sophos-alert-123",
  "alert_level": 5,
  "threat_type": "malware",
  "threat_name": "Trojan.X",
  "source_host": "workstation-01",
  "source_ip": "192.168.1.100",
  "timestamp": "2026-04-27T10:00:00Z"
}

Mapping DSL transforma em schema canônico:

timestamp → timestamp (ISO 8601)
severity → map(alert_level, {1: "low", 5: "critical"})
title → threat_name
source → source_host
...
