Versão: 1.7 (legada)

Adicionar novo vendor

Para engenheiros que querem adicionar suporte a novo vendor (ex: CrowdStrike, Fortinet, etc).

Público: Engineer+. Requer conhecimento Python + FastAPI.

Arquitetura pluggable​

CentralOps tem registry pluggable de providers e collectors.

backend/app/providers/
  ├─ base.py              # BaseProvider (interface)
  ├─ sophos/
  │  ├─ provider.py       # SophosProvider (implementação)
  │  └─ __init__.py
  ├─ wazuh/
  │  ├─ provider.py
  │  └─ __init__.py
  └─ seu_novo_vendor/
     ├─ provider.py       # VocêProvider (new)
     └─ __init__.py

backend/app/collectors/
  ├─ base.py              # BaseCollector (interface)
  ├─ vendors/
  │  ├─ sophos.py
  │  ├─ wazuh.py
  │  └─ seu_novo_vendor.py  # (new)
  └─ registry.py          # CollectorRegistration

2.2 Implementar SeuNovoCollector​

# backend/app/collectors/vendors/seu_novo_vendor.py

from __future__ import annotations

from typing import Any, AsyncIterator, Dict
from datetime import datetime, timedelta, timezone
import logging

from ..base import BaseCollector
from ..registry import CollectorRegistration, register

logger = logging.getLogger(__name__)


class SeuNovoAlertsCollector(BaseCollector):
    platform = "seu_novo_vendor"
    stream = "alerts"
    event_type = "seu_novo_vendor.alert"
    
    async def collect(self) -> AsyncIterator[Dict[str, Any]]:
        """Collect alerts from API."""
        # 1. Get cursor (where we left off)
        cursor = self.ctx.cursor or {}
        from_ts = cursor.get("from_ts") or (
            datetime.now(timezone.utc) - timedelta(minutes=5)
        ).isoformat()
        
        # 2. Authenticate & call API
        api_key = self.ctx.integration.metadata.get("api_key")
        # TODO: add token refresh if OAuth
        
        url = "https://api.seu_vendor.com/alerts"
        
        async with self.ctx.session.get(
            url,
            headers={"Authorization": f"Bearer {api_key}"},
            params={"from": from_ts},
        ) as resp:
            resp.raise_for_status()
            payload = await resp.json()
        
        # 3. Yield events
        for alert in payload.get("alerts", []):
            yield {
                "alert_id": alert["id"],
                "title": alert["title"],
                "severity": alert["severity"],
                "timestamp": alert["created_at"],
                "raw": alert,
            }
            
            # 4. Update cursor for next run
            self.ctx.cursor = {"from_ts": alert["created_at"]}


# Register collector
register(CollectorRegistration(
    platform="seu_novo_vendor",
    stream="alerts",
    collector_cls=SeuNovoAlertsCollector,
    schedule=timedelta(minutes=5),
    queue="collect.priority",
    task_name="collectors.collect_vendor_logs_priority",
))

Passo 5: Teste​

5.1 Unit test​

# tests/collectors/test_seu_novo_vendor.py

import pytest
from backend.app.collectors.vendors.seu_novo_vendor import SeuNovoAlertsCollector
from backend.app.db.models import Integration

@pytest.mark.asyncio
async def test_seu_novo_alerts_collect(mocker):
    # Mock API response
    mocker.patch(
        "httpx.AsyncClient.get",
        return_value=mocker.MagicMock(
            json=lambda: {
                "alerts": [
                    {
                        "id": "alert-1",
                        "title": "Test",
                        "severity": "high",
                        "created_at": "2026-04-27T10:00:00Z",
                    }
                ]
            }
        ),
    )
    
    # Create integration
    integration = Integration(
        platform="seu_novo_vendor",
        metadata={"api_key": "test_key"},
    )
    
    # Collect
    collector = SeuNovoAlertsCollector(integration)
    events = [e async for e in collector.collect()]
    
    assert len(events) == 1
    assert events[0]["alert_id"] == "alert-1"

Boas práticas​

  1. Erro handling: sempre catch e log erros (não crash collector).
  2. Rate limiting: respeite HTTP 429, implemente backoff exponencial.
  3. Idempotência: usar message_id do vendor para dedup.
  4. Cursor: persistir em Redis para delta sync (não re-ingerir).
  5. Type safety: use TypedDict ou Pydantic para schema.
  6. Async: collectors são assíncrono. Use async/await, nunca sync HTTP.
  7. Logging: log início/fim de coleta, erros com contexto.
  8. Tests: mínimo 1 teste parametrizado com mock.

Checklist final​

  • Provider: herda de BaseProvider, implementa test_connection().
  • Collector: herda de BaseCollector, implementa collect() async.
  • Registro: ambos registrados via registry.
  • Teste: unit test com mock + integration test manual.
  • Documentação: arquivo markdown com guia passo-a-passo.
  • Permissions: se necessário, adicione em auth.py.
  • Mapping padrão: crie defaults JSON para início rápido.

Deploy​

Após merge:

  1. Build imagem: docker compose build.
  2. Start: docker compose up.
  3. Verifique health: docker compose logs collector-beat | grep registered.
  4. Seu vendor aparece em dropdown Integrações.
