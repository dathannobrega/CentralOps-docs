Wazuh Integration

Integre seu Wazuh Manager (API) e Wazuh Indexer (Elasticsearch) para bidirecional com CentralOps.

Público: Admin (criar integração), Engineer+ (consultar).

O que você precisa​

  • ✓ Wazuh Manager: URL + credenciais (Basic Auth).
  • ✓ Wazuh Indexer: URL + credenciais (Basic Auth).
  • ✓ Certificado TLS (se Indexer usa SSL, opcional mas recomendado).

Pré-requisitos​

Manager credentials​

Acesse seu Wazuh:

URL: https://wazuh-manager.seu-dominio.com

Crie usuário dedicado (ou use admin):

Usuário: centralops (recomendado: novo usuário)
Senha: senha_forte_aqui

Role recomendada: "Agents" + "Read" (mínimo para ler agents/índices).

Indexer credentials​

Acesso via Elasticsearch API (padrão port 9200):

URL: https://wazuh-indexer.seu-dominio.com:9200
Usuário: indexer_user (ou admin)
Senha: indexer_password

Permissões: read em índices wazuh-alerts-*.

Passo 1: Obter URLs e credenciais​

1.1 Manager​

# Terminal onde Wazuh está rodando
curl -k -u admin:password https://wazuh-manager:55000/api/version
# Se responde 200 OK, credenciais estão corretas

Anote:

  • Manager URL: https://wazuh-manager:55000 (ou seu domínio)
  • Manager username: admin
  • Manager password: senha

1.2 Indexer​

curl -k -u admin:password https://wazuh-indexer:9200/
# Se responde com cluster info, credenciais OK

Anote:

  • Indexer URL: https://wazuh-indexer:9200
  • Indexer username: admin
  • Indexer password: senha

1.3 Certificado TLS (opcional)​

Se Wazuh usa CA customizada (não Let's Encrypt):

# Obter CA cert do Wazuh
docker exec wazuh cat /etc/ssl/certs/wazuh-ca.pem > wazuh-ca.pem

# Copiar para CentralOps
docker cp wazuh-ca.pem centralops:/certs/wazuh-ca.pem

Passo 2: Criar integração em CentralOps​

2.1 Abrir CentralOps​

Vá a Administração → Integrações.

Clique + Adicionar integração.

2.2 Selecionar plataforma​

Dropdown → Wazuh.

Clique Próximo.

2.3 Preencher credenciais — Manager​

Nome:                   Wazuh - Production
Manager URL:            https://wazuh-manager:55000 (ou seu domínio)
Manager API username:   admin (ou user dedicado)
Manager API password:   senha_aqui
Verify SSL:             ✓ (marque, recomendado)

2.4 Preencher credenciais — Indexer​

Indexer URL:            https://wazuh-indexer:9200
Indexer username:       admin (ou user dedicado)
Indexer password:       senha_aqui
Custom CA cert:         (deixe vazio se Let's Encrypt ou auto-signed)
                        OU cole path: /certs/wazuh-ca.pem

2.5 Testar conexão​

Clique Testar conexão.

Sistema valida:

  • Conectividade a Manager (API works).
  • Conectividade a Indexer (Elasticsearch responde).
  • Credenciais corretas (200 OK).
  • SSL cert válido.

Se OK: "✓ Ambas as conexões bem-sucedidas".

2.6 Salvar​

Clique Salvar.

Passo 3: Verificar integração​

3.1 Health check​

Vá a Pipeline Health → procure integração Wazuh.

Status deve ser "Ativo" e verde.

3.2 Consultar alertas​

Wazuh pode ser usado de 2 formas:

A) Coletar alertas de Wazuh (ingress):

Vá a Busca → filtro platform:wazuh.

Você vê alertas já presentes no Wazuh Indexer.

B) Enviar alertas para Wazuh (egress):

Isso é automático — todos os eventos normalizados são enviados via Syslog ao Wazuh Manager.

Vá a Wazuh Kibana → índice wazuh-alerts-* → procure por namespace _centralops.

Troubleshooting​

"Connection refused"​

Erro: "Failed to connect to Manager/Indexer"
Ação:
  1. Verificar se Wazuh está rodando: curl -k https://wazuh-manager:55000
  2. Firewall bloqueando? Testar acesso de CentralOps container
  3. URL correta? (com protocol https://)

"401 Unauthorized"​

Erro: "Invalid credentials"
Ação:
  1. Username/password corretos em Wazuh?
  2. Usuário tem permissão? (roles: agents, read)
  3. Senha mudou? Gere novo token em Wazuh

"SSL certificate error"​

Erro: "Certificate verification failed" ou "CA cert not found"
Ação:
  1. Se auto-signed: marque "Verify SSL" = OFF (menos seguro)
  2. Se CA custom: copiar CA cert para /certs/wazuh-ca.pem
  3. Testar: curl -k --cacert /certs/wazuh-ca.pem https://wazuh-indexer:9200

"Indexer quota exceeded"​

Erro: "429 Too Many Requests" ou "Cluster is yellow/red"
Ação:
  1. Wazuh Indexer está com espaço? Verificar storage
  2. Limpar índices antigos: DELETE wazuh-alerts-2026-04-01
  3. Aumentar shards do Indexer (Wazuh admin task)
  4. Escalar para Wazuh suporte

Dados coletados (via Indexer query)​

CentralOps executa query no Indexer para alertas:

GET wazuh-alerts-*/_search
{
  "query": {
    "range": {
      "@timestamp": { "gte": "now-5m" }
    }
  }
}

Resultado: eventos com schema Wazuh:

{
  "@timestamp": "2026-04-27T10:00:00Z",
  "data": {
    "severity": 7,
    "title": "Root login attempted",
    ...
  },
  "agent": {
    "id": "001",
    "name": "web-server-01"
  }
}

Mapping DSL transforma em schema canônico (timestamp, severity, title, etc).

Egress (envio para Wazuh)​

Automático — todos os eventos normalizados são despachados:

Protocolo: Syslog RFC 5424 (TCP TLS)
Porta:     6514 (padrão Wazuh)
Formato:   JSON lines (cada evento é 1 linha)

No Wazuh Indexer, procure por namespace _centralops:

{
  "_centralops": {
    "integration_id": 42,
    "platform": "sophos",
    "collected_at": "2026-04-27T10:00:00Z"
  },
  "severity": "high",
  "title": "Malware detected",
  ...
}

Casos de uso​

Caso 1: Centralizar múltiplos Wazuh Managers​

Se você tem Wazuh em 2 regiões:

  1. Crie integração Wazuh para região 1.
  2. Crie integração Wazuh para região 2.
  3. CentralOps agrega alertas de ambas.
  4. Dashboard mostra painel único.

Caso 2: Normalizar e re-enviar​

Alguns clientes enviam dados para Wazuh em formato inconsistente.

  1. Integração Wazuh puxa alertas brutos.
  2. Mapping DSL normaliza campos.
  3. Dispatcher re-envia para Wazuh Indexer.
  4. Kibana agora tem dados consistentes.
