Visão geral de integrações

CentralOps conecta-se a múltiplos vendors de segurança via APIs, coleta eventos, os normaliza (CML) e roteia pro destino configurado.

Pluggable nos dois lados. A arquitetura é pluggable tanto na entrada (registry de providers — adicionar um vendor é registrar uma classe BaseProvider) quanto na saída (_Target Protocol — Wazuh hoje, mas você implementa Splunk, Elastic, S3, Sentinel ou Kafka como destino adicional). Ver Adicionar Vendor.

Público: Admin+ (criar/deletar). Viewer+ (ver status).

Vendors suportados​

1. Sophos Central​

Plataforma de proteção de endpoints (EDR/XDR) para Windows, macOS, Linux.

Streams:

  • alerts — detecções de malware, comportamento suspeito.
  • cases — casos de XDR (Managed Threat Response).

Autenticação: OAuth 2.0 (Client Credentials).

Leia Sophos Integration.

2. Wazuh (SIEM nativo)​

Integração bidirecional com seu SIEM Wazuh.

Função:

  • Ingress: puxar alertas de Wazuh Indexer (busca por vendor/severidade).
  • Egress: enviar eventos normalizados de volta (pipeline padrão).

Autenticação: Basic Auth (Manager + Indexer).

Leia Wazuh Integration.

Em desenvolvimento​

Microsoft Defender for Endpoint e NinjaOne (RMM) têm collectors parciais em backend/app/collectors/vendors/, mas ainda não possuem provider classe em backend/app/providers/registry.py. Isso significa que o sistema de coleta está pronto para ingestão, mas a integração não é utilizável via API até que a camada de provider seja implementada. Estes vendors serão adicionados à lista de "Vendors suportados" assim que o provider estiver completo.

Fluxo de integração​

1. Admin cria integração (Adm → Integrações)
   └─ Preenche credenciais (client_id, secret, etc)
   └─ Seleciona plataforma (sophos, defender, etc)

2. CentralOps valida credenciais (test_connection)
   └─ Se OK: integração entra estado "active"
   └─ Se erro: estado "error", mostra mensagem

3. Collector Beat agenda coleta
   └─ RedBeat insere schedule no Redis
   └─ Próxima execução: 1-5 min conforme vendor

4. Collector workers puxam eventos
   └─ Delta sync (cursor persistido em Redis)
   └─ Dedup por vendor message_id
   └─ Armazenam em PostgreSQL (RawEvent)

5. Motor de normalização transforma
   └─ Mapping DSL aplica regras
   └─ Se sucesso: NormalizedEvent criado
   └─ Se erro: QuarantineEvent criado

6. Dispatcher envia ao Wazuh
   └─ Syslog TCP/TLS (port 6514)
   └─ Ou arquivo para filebeat

7. Dashboard/Alertas/Busca consultam dados
   └─ Viewer vê eventos normalizados
   └─ Engineer pode editar mappings e reprocessar

Health checks​

Cada integração tem health status:

StatusSignificado
VerdeColeta ativa, última execução < 5 min, sem erros.
AmareloColeta mais lenta, ou erros esporádicos (429 rate limit).
VermelhoParada > 10 min, ou erro crítico (401 unauthorized).

Admin ve status em Administração → Integrações → detail da integração.

Ações por usuário​

Viewer​

  • Ver lista de integrações.
  • Ver health status.
  • Ver última coleta.

Operator​

  • Tudo do Viewer +
  • Pausar/retomar coleta.
  • Testar conexão.

Engineer​

  • Tudo do Operator +
  • (Nada adicional para integrações — foco em mappings)

Admin​

  • Tudo +
  • Criar integração.
  • Editar credenciais.
  • Deletar integração.
  • Configurar schedule de coleta.

Gerenciamento de credenciais​

Todas as credenciais são:

  • Criptografadas em repouso (AES-256-GCM).
  • Nunca exibidas na UI (segurança).
  • Rotadas automaticamente (OAuth refresh tokens).

Admin pode resetar credencial sem perder coleta:

  1. Vá a Administração → Integrações → clique integração.
  2. Clique Editar credenciais.
  3. Preencha novo secret (ex: novo Client Secret de Sophos).
  4. Clique Salvar.
  5. Sistema valida, substitui se OK.

Escalação​

Se integração fica vermelha:

  1. Verificação rápida: Clique health → vê último erro.
  2. Teste manual: botão "Testar conexão".
  3. Credenciais expiradas?: Reset em "Editar credenciais".
  4. Rate limit?: Aumentar intervalo de coleta.
  5. Vendor offline? Aguardar ou escalar para suporte do vendor.

Para runbook completo, veja Dispatcher.
