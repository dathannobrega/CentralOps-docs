Versão: 1.7 (legada)

Quickstart: Sophos em 15 minutos

Seu objetivo: conectar Sophos Central, coletar primeiros alertas, vê-los no dashboard e na quarentena.

Pré-requisitos​

  • ✓ Você tem conta admin em CentralOps.
  • ✓ Você tem acesso administrativo a Sophos Central (sua organização).
  • ✓ Você gerou Client ID e Client Secret (na UI do Sophos, Settings > API Credentials).

Não tem? Siga: Sophos Integration Guide.

Passo 1: Obter credenciais Sophos (5 min)​

No console Sophos Central:

  1. Vá a Settings > API Credentials.
  2. Clique Create (ou Add se já existe).
  3. Nome: "CentralOps".
  4. Permissões: marque ao menos alerts:read, cases:read (se usar XDR).
  5. Copie Client ID e Client Secret — salve em local seguro.

Também anote:

  • Region: veja em sua URL Sophos (https://api-eu.central.sophos.com = region EU).
  • Tenant ID: está em Settings > Organization (campo "ID").

Passo 2: Criar integração no CentralOps (5 min)​

  1. Abra CentralOps. Vá a Administração → Integrações.
  2. Clique + Adicionar integração.
  3. Selecione Sophos.
  4. Preencha:
    • Nome: "Sophos - Acme Corp" (identificação).
    • Client ID: cola do passo 1.
    • Client Secret: cola do passo 1.
    • Tenant ID: cola do passo 1.
    • Region: selecione (US, EU, APAC, etc).
    • Organization: deixe em branco ou selecione se multi-tenant (admin só).
  5. Clique Testar conexão. Deve responder "OK".
  6. Clique Salvar.

Sistema agora:

  • ✓ Armazenou credenciais cifradas.
  • ✓ Validou credenciais OAuth.
  • ✓ Agendou primeira coleta (próximos 1-2 min).

Passo 3: Aguardar primeira coleta (2 min)​

Integração entra em estado "Aguardando primeira coleta".

Aguarde 2-3 minutos. Celery Beat dispara task de coleta. Se Sophos retorna alertas, você verá:

  • Estado muda para "Ativo".
  • Health muda para verde.
  • Eventos aparecem em quarentena (se mapping não existe) ou são normalizados direto.

Passo 4: Verificar alertas (3 min)​

Opção A: Dashboard​

Vá a Operação → Dashboard.

  • Veja card "Últimos eventos normalizados".
  • Se tiver alertas críticos, verá neste card.

Opção B: Quarentena (mais provável na primeira coleta)​

Vá a Operação → Quarentena.

  • Lista eventos que falharam parsing ou validação.
  • Clique num evento para expandir e ver raw JSON + erro.

Opção C: Integração detail​

Vá a Administração → Integrações → clique na integração.

  • Veja métricas de saúde (última coleta, eventos processados).
  • Se houver erro, verá mensagem.

Passo 5: Revisar mapping padrão (opcional)​

Se eventos estão em quarentena, é porque mapping não existe ou está errado.

  1. Vá a Mappings → Normalization → Overview.
  2. Abra mapping sophos.alert (ou crie um se não existe).
  3. Revise regras DSL. Veja Cookbook para sintaxe.
  4. Edite se necessário (ex: field threat → severity).
  5. Salve. Versão anterior fica disponível para rollback.

Após salvar:

  • Novos eventos usam novo mapping.
  • Você pode reprocessar eventos em quarentena se quiser.

Status esperado após 15 min​

ItemStatus
Integração✓ Ativa, com health verde
Eventos✓ Chegando a cada 1 min (~60 por ciclo)
Quarentena✓ Vazia (se mapping está correto) ou com poucos erros
Dashboard✓ Mostra "Últimos eventos normalizados"

Próximos passos​

  • Dashboard — leia sobre KPIs e alertas.
  • Quarentena — entenda triage e reprocessamento.
  • Mappings — customize regras de normalização.
  • Mais integrações — adicione Defender, NinjaOne, Wazuh.

Troubleshooting rápido​

"Integração não coleta"

# No container, verificar logs do Beat
docker compose logs collector-beat | tail -50
# Procure por erros de autenticação ou taxa limite (429)

"Erro 401 Unauthorized"

Sophos rejeitou credenciais. Verifique:

  • Client ID e Client Secret corretos?
  • Tenant ID correto?
  • Region correta?
  • Credenciais ainda ativas em Sophos Central?

"Quarentena cheia de eventos"

Significa mapping não existe ou está rejeitando fields. Veja Troubleshooting.

"Sem eventos em Dashboard"

Aguarde 5-10 min (primeira coleta é lenta). Se continuar vazio, verifique integração em quarentena acima.
