Inventory

Assets (hosts, endpoints, devices) descobertos automaticamente de integrações.

Público: Viewer+.

Dados de origem​

Cada integração contribui com assets:

IntegraçãoAssets coletados
SophosEndpoints gerenciados (Windows, Mac, Linux).
DefenderDevices registrados no tenant.
NinjaOneMachines monitoradas.
WazuhAgents registrados.

Dedupe automático: se mesmo hostname aparece em múltiplas integrações, mesclado por vendor_id único.

Interface​

Lista de assets​

Colunas padrão:

ColunaDescrição
HostnameNome do device.
IPEndereço IPv4/IPv6.
PlataformaOS (Windows, macOS, Linux, etc).
VendorIntegração de origem.
StatusLast seen (online, offline).
AçãoVer histórico de alertas.

Filtros​

  • Plataforma: Windows, macOS, Linux, etc.
  • Vendor: Sophos, Defender, NinjaOne.
  • Status: Online, Offline, Unknown.
  • Busca: hostname, IP.

Workflow​

1. Procurar asset por hostname​

Campo busca: "servidor-web-01"
Clique: Buscar
Resultado: asset encontrado (ou não encontrado)

2. Ver detalhes​

Clique em um asset:

Painel lateral abre com:
- Hostname
- IP(s)
- Plataforma
- Última coleta
- Agentes running (Wazuh agent, Sophos, etc)
- Alertas recentes (últimas 24h)

3. Ver histórico de alertas daquele asset​

Dentro do detalhe, clique Ver alertas → redireciona para Alertas filtrado por source_host.

Resultado: todos os eventos críticos desse asset

4. Correlacionar entre integrações​

Asset aparece em múltiplas integrações?

Exemplo: "servidor-web-01" em Sophos, Defender, Wazuh
Clique em asset → vê em qual integração foi descoberto primeiro
Compara status entre vendors (offline em Sophos mas online em Defender?)

Discrepâncias podem indicar:

  • Agente está desatualizado.
  • Comunicação com vendor lenta.
  • Device isolado de parte da rede.

Casos de uso​

IR: asset está comprometido?​

  1. Vá a Inventory.
  2. Busque hostname (ex: "workstation-xyz").
  3. Clique → expande detalhe.
  4. Vê "Alertas recentes" → pula para Alertas com esse asset.
  5. Revisa timeline: quando foi primeiro alerta? Que tipo?

Vulnerabilidade conhecida: qual assets afetados?​

  1. Busca em Busca avançada: title:"CVE-2026-xxxx".
  2. Nota assets afetados.
  3. Vem para Inventory → filtro plataforma (ex: Windows).
  4. Vê quais estão offline (talvez já foram patched).
  5. Filtra online → lista de remediação.

Cobertura de coleta​

  1. Vá a Inventory.
  2. Filtro: Vendor = Sophos.
  3. Vê N assets cobertos por Sophos.
  4. Compara com número esperado (ex: "temos 200 endpoints, CentralOps vê 195").
  5. Se discrepância, cheque integração saúde ou credenciais Sophos expiradas.

Limitações​

  • Sync delay: assets são atualizados a cada 5-15 min (conforme schedule de integração). Novo device demora até 15 min para aparecer.
  • Offline fallback: se integração parou, assets ficam com last_seen antigo. Não é deletado (serve para histórico).
  • IP privado: alguns vendors omitem IP (Sophos em geral mostra). CentralOps não enriquece com DNS reverso (roadmap).

Próximos passos​

  • Quer timeline de um asset? Vá a Alertas com filtro source_host.
  • Asset offline suspeita? Vá a Pipeline Health para checar integração.
  • Cobertura baixa? Vá a Administração → Integrações.
