Alertas

Lista de eventos normalizados com busca, filtros e priorização.

Público: Viewer+.

Interface​

Filtros​

Por severidade:

  • Critical, High, Medium, Low — badge colorida.
  • Selecione um para ver só eventos daquela severidade.

Por data:

  • Picker de calendário: "últimas 24h", "últimos 7 dias", custom range.
  • Padrão: últimas 24h (retenção máxima: 7 dias).

Por plataforma:

  • Dropdown: Sophos, Defender, NinjaOne, etc.
  • Multi-select possível.

Por palavra-chave:

  • Campo de texto livre (busca full-text em título + descrição).

Colunas visíveis​

ColunaDescrição
TimestampQuando o evento foi criado (timezone UTC).
PlataformaVendor de origem (sophos, defender, ninjaone).
SeveridadeBadge colorida (critical, high, medium, low).
TítuloDescrição curta (ex: "Trojan.X detectado").
AçãoExpand para ver JSON completo, copiar ID, etc.

Workflow​

1. Buscar eventos críticos​

  1. Filtro Severidade = Critical.
  2. Filtro Data = "últimas 24h".
  3. Clique enter ou botão Buscar.
  4. Resultado mostra N eventos críticos.

2. Revisar detalhe de 1 evento​

  1. Clique no evento na lista.
  2. Painel lateral abre com JSON completo.
  3. Veja:
    • severity: avaliação de risco.
    • title: resumo.
    • source: que integração coletou.
    • timestamp: quando.
    • raw_data: todos os campos (pode ser longo).

3. Achar padrão de ataque​

  1. Filtro Plataforma = Sophos.
  2. Filtro Palavra-chave = "ransomware" ou nome de IOC.
  3. Resultado mostra todos os eventos correspondentes.
  4. Nota: padrão é descendente por timestamp (mais recente primeiro).

Se múltiplos eventos similares em poucos minutos → possível ataque em andamento.

4. Drill-down em asset​

  1. Clique em evento → expande detalhe.
  2. Procure source_host ou affected_asset.
  3. Copie hostname/IP.
  4. Vá a Inventory → busque por hostname.
  5. Veja histórico de alertas daquele asset.

Comportamento de busca​

  • Combinação AND: filtros combinam com AND (todos precisam match).
  • Paginação: mostra 50 eventos por página. Navegue com botões.
  • Ordem: padrão descendente por timestamp (mais recente = primeiro).
  • Retenção: apenas últimos 7 dias estão disponíveis. Para dados históricos, consulte Wazuh Indexer ou backup.

Casos de uso​

CasoPassos
Houve ataque crítico?Filtro: Severity=Critical, Data=24h. Se > 0, inicia IR.
Qual vendor tem mais alertas?Filtro por plataforma, compare contagens.
Falsos positivos?Filtro: Platform=X, Palavra-chave=Y. Se padrão repetido, revise mapping.
Asset atacado recentemente?Busca por hostname/IP. Veja timeline de eventos.

Limitações​

  • Dashboard mostra 7 dias max. Wazuh Indexer pode ter dados mais antigos.
  • Não há ação possível desta lista (read-only). Para discardar/reprocessar, vá a Quarentena.
  • Busca full-text é básica (substring). Para regex, use Busca avançada.

Próximos passos​

  • Evento em quarentena? Vá a Quarentena.
  • Asset relacionado? Vá a Inventory.
  • Busca mais avançada? Vá a Busca.
