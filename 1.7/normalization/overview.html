CML — CentralOps Mapping Language

CML (CentralOps Mapping Language) é a DSL declarativa do CentralOps para normalizar eventos brutos de vendors de segurança em OCSF (Open Cybersecurity Schema Framework). É a linguagem que vocês usam pra escrever regras de mapeamento — versionada, com diff, dry-run, rollback e drift detection nativos.

O que faz​

Quando uma detecção do Sophos chega como JSON, ela tem nomes de campos e tipos específicos do vendor. O motor CML lê suas regras de mapeamento e produz um evento OCSF padronizado que suas ferramentas de SOC entendem.

CML é declarativa (você descreve o quê, não o como), versionada no banco (cada commit gera nova versão append-only), e auditável (dry-run contra sample reservoir antes de commit + drift detection automática de campos novos do vendor).

Entrada: Evento bruto do vendor (forma opaca, semântica do vendor)
Mapeamento: Regras DSL que você cria
Saída: Evento normalizado em OCSF (normalized.*)

Início Rápido — Escreva Seu Primeiro Mapeamento​

Um mapeamento é um documento JSON com preprocess (lista de operações, opcional) e rules (lista de regras). Cada regra diz "pegue um valor daqui, transforme-o e escreva ali".

{
  "preprocess": [],
  "rules": [
    // Metadados OCSF obrigatórios
    { "target": "normalized.class_uid", "const": 2004 },
    { "target": "normalized.class_name", "const": "Detection Finding" },

    // Mapear um campo do vendor para um campo OCSF
    {
      "target": "normalized.time",
      "source": "eventTime",          // extrair do evento bruto
      "type_cast": "iso_to_epoch",    // converter ISO-8601 para segundos
      "required": true                // falhar se ausente
    },

    // Use fallback se o campo principal estiver ausente
    {
      "target": "normalized.finding_info.title",
      "source": "ruleName",
      "fallback_source": ["description"],
      "default": "Unknown Detection"
    }
  ]
}

Pronto. Cada regra tem um target (caminho OCSF) e uma source (expressão JMESPath) mais transformações opcionais.

Recursos da DSL​

NecessidadeRecurso
Mapeamento de campo básicoregra escalar com target + source
Analisar JSON aninhado antes de mapearadicione op em preprocess (ex: json_parse)
Múltiplos observáveis (IPs, emails, hashes)regra com kind: "array_builder"
Regras condicionaispredicados when: { exists: ... }
