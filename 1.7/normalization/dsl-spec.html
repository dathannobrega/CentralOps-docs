Especificação DSL

O DSL de normalização é um formato declarativo para mapear campos de vendor a OCSF.

Formato DSL (v2)​

Um mapeamento v2 é um objeto JSON com duas seções:

{
  "preprocess": [
    { "op": "json_parse", "source": "processedData", "target": "_processed", "tolerant": true }
  ],
  "rules": [
    { "target": "...", "source": "_processed.field", ... }
  ]
}

Recursos:

  • Bloco preprocess — extrair e transformar dados brutos uma vez antes das regras executarem
  • fallback_source — tentar múltiplas sources na ordem
  • when — regras condicionais (executar apenas se o predicado corresponder)
  • kind: "array_builder" — construir arrays de observáveis de múltiplas sources
  • expected_always_default — marcar regras de espaço reservado intencional

Estrutura de Regra​

Toda regra é um objeto JSON com estes campos:

CampoTipoObrigatório?
targetstringsim
sourceJMESPathnão*
constanynão*
defaultanynão
pre_caststringnão
value_mapobjectnão
type_caststringnão
requiredbooleannão
fallback_sourcelist[string]não
whenobjectnão
expected_always_defaultbooleannão

* Ou source ou const é obrigatório; são mutuamente exclusivos.

target (obrigatório)​

Semântica: Onde escrever o resultado no evento normalizado.

Tipo: String, caminho separado por ponto.

Exemplos:

"target": "normalized.class_uid"
"target": "normalized.finding_info.title"
"target": "normalized.device.name"

Regras:

  • Não deve ser vazio ou conter apenas pontos.
  • Em v2 com pré-processamento, targets em rules NÃO devem começar com _ (reservado para saídas de pré-processamento).

source (obrigatório: source ou const)​

Semântica: Expressão JMESPath para extrair um valor do evento bruto.

Tipo: String (sintaxe válida de JMESPath).

Exemplos:

"source": "severity"                    // Busca de campo simples
"source": "device.name"                 // Campo aninhado
"source": "createdAt || raisedAt"       // Fallback no próprio JMESPath
"source": "tags | length(@)"            // Pipes e filtros
"source": "_processed.parsedAlert.fields.clientIp"  // v2: de dados pré-processados

Comportamento:

  • Se a expressão retorna null, move para a etapa default.
  • Array vazio [] conta como um valor válido, não null.
  • Sintaxe JMESPath inválida no tempo de compilação do mapeamento lança MappingDefinitionError.

Referência cruzada: Tutorial JMESPath

const (obrigatório: source ou const)​

Semântica: Valor literal (ignora extração de source).

Tipo: Qualquer tipo JSON.

Exemplos:

"const": 2004
"const": "Detection Finding"
"const": true
"const": null
"const": [ "tag1", "tag2" ]

Comportamento:

  • Valor literal é escrito como está.
  • Pula a etapa source mas ainda aplica default, pre_cast, value_map, type_cast.
  • Útil para metadados OCSF que são iguais para todos os eventos de um vendor.

default (opcional)​

Semântica: Valor de fallback se a source resolveu para null.

Tipo: Qualquer tipo JSON.

Exemplo:

{
  "target": "normalized.message",
  "source": "description",
  "default": null
}

Comportamento:

  • Aplicado imediatamente após resolução de source/const.
  • Se o valor é null e default existe, o valor torna-se default.
  • Se o valor não é null, default é ignorado.

Timing no pipeline: Primeiro — antes de pre_cast, value_map, type_cast.

pre_cast (opcional)​

Semântica: Conversão de tipo aplicada ANTES da busca do value_map.

Tipo: String (nome de um cast registrado).

Exemplo:

{
  "target": "normalized.severity_id",
  "source": "severity",
  "pre_cast": "to_str",       // converter int em string
  "value_map": {
    "high": 4,
    "medium": 3,
    "low": 2
  }
}

Caso de uso: Vendor envia severity como int (0-10) mas suas chaves de busca são strings ("high", "medium").

Timing no pipeline: Segundo — após default, antes de value_map.

Casts suportados: Veja Referência de Operadores — Type Casts.

value_map (opcional)​

Semântica: Tabela de busca chave-valor (case-insensitive para chaves string).

Tipo: Object (dicionário).

Exemplo:

{
  "target": "normalized.severity_id",
  "source": "severity",
  "value_map": {
    "critical": 5,
    "high": 4,
    "medium": 3,
    "low": 2,
    "info": 1
  },
  "default": 0
}

Comportamento:

  • Valor de entrada é buscado no mapa.
  • Se a chave é encontrada, usa o valor mapeado; caso contrário, o valor passa inalterado.
  • Chaves string são case-insensitive (chave "high" corresponde ao valor "HIGH").
  • Se o valor não está no mapa e não há default, o valor não mapeado é retornado como está.

Timing no pipeline: Terceiro — após pre_cast, antes de type_cast.

type_cast (opcional)​

Semântica: Conversão de tipo (aplicada por último).

Tipo: String (nome de um cast registrado).

Exemplos:

{ "target": "normalized.time", "source": "eventTime", "type_cast": "iso_to_epoch" }
{ "target": "normalized.severity", "source": "level", "type_cast": "uppercase" }

Timing no pipeline: Quarto (último) — após value_map.

Casts suportados: Veja Referência de Operadores — Type Casts.

required (opcional, padrão: false)​

Semântica: Se verdadeiro, falhar todo o evento se o valor final for null.

Tipo: Boolean.

Comportamento:

  • Se required: true e o valor resolvido for null, lança MappingRequiredFieldError.
  • Pipeline intercepta o erro e envia o evento para quarentena com error_kind: "map".
  • Use para campos OCSF que devem existir (ex: time, finding_info.uid).

Exemplo:

{
  "target": "normalized.time",
  "source": "createdAt",
  "type_cast": "iso_to_epoch",
  "required": true  // Falhar se createdAt estiver ausente
}

fallback_source (v2 somente, opcional)​

Semântica: Lista de expressões JMESPath adicionais tentadas se a source primária resolver para null.

Tipo: Lista de strings (expressões JMESPath).

Exemplo:

{
  "target": "normalized.finding_info.title",
  "source": "ruleTitle",
  "fallback_source": ["attackType", "description"],
  "default": "Unknown"
}

Comportamento:

  • A source primária é tentada primeiro.
  • Se retorna null, fallback_source[0] é tentada.
  • Se ainda for null, fallback_source[1] é tentada, etc.
  • Se todos os fallbacks forem null, default é usado.
  • Todas as expressões de fallback devem usar a mesma raiz que a source primária (ambas raw ou ambas _extracted).

Timing no pipeline: Parte da resolução de source; antes de default.

Referência cruzada: Receita 3 em Cookbook.

when (v2 somente, opcional)​

Semântica: Portão de predicado — regra executa apenas se a condição for verdadeira.

Tipo: Object (dicionário de predicado).

Comportamento:

  • Se o predicado avalia como falso, a regra é PULADA inteiramente (target não é escrito).
  • Semanticamente diferente de escrever null ou um default.

Exemplo:

{
  "target": "normalized.actor.user.email_addr",
  "source": "mailFrom",
  "when": { "exists": "mailFrom" }
}

Predicados: Veja Referência de Operadores — Predicados.

Timing: Avaliado antes da execução da regra.

Referência cruzada: Receita 6 em Cookbook.

expected_always_default (v2 somente, opcional, padrão: false)​

Semântica: Flag de diagnóstico — marca esta regra como intencionalmente sempre produzindo um valor padrão.

Tipo: Boolean.

Comportamento:

  • NÃO muda a execução da regra.
  • Usado pelo dry-run para suprimir avisos "100% default" para gaps conhecidos.
  • Quando verdadeiro e dry-run encontra a regra sempre usa default, não é sinalizado como um problema.

Exemplo:

{
  "target": "normalized.src_endpoint.ip",
  "source": "sourceIp",
  "default": null,
  "expected_always_default": true  // Este vendor nunca fornece sourceIp
}

Bloco de Pré-processamento (v2 somente)​

O array preprocess executa uma vez por evento antes de qualquer regra executar, populando o namespace _extracted.

Forma:

{
  "preprocess": [
    {
      "op": "json_parse",
      "source": "processedData",
      "target": "_processed",
      "tolerant": true
    }
  ],
  "rules": [ ... ]
}

Campos de pré-processamento​

CampoTipoObrigatório?
opstringsim
sourceJMESPathsim
targetstringsim
tolerantbooleannão

op — Nome do operador de pré-processamento ("json_parse" por enquanto).

source — Expressão JMESPath para extrair do evento bruto (NÃO de _extracted).

target — Onde armazenar resultado; DEVE começar com _ (namespace reservado).

tolerant (padrão: false) — Se verdadeiro, erros silenciosamente produzem null em vez de lançar.

Ops suportados: Veja Referência de Operadores — Preprocess Ops.

Ordem de Operações​

Para cada regra (v1 & v2):

  1. Resolução de Source/Const: Extrair valor via JMESPath ou usar literal.
  2. Default: Se null, usar valor padrão.
  3. Pre-cast: Aplicar cast opcional (antes de busca).
  4. Value map: Aplicar busca opcional.
  5. Type-cast: Aplicar cast opcional (após busca).
  6. Escrever: Atribuir ao caminho de target.

Exemplo de rastreamento:

Evento bruto:

{
  "severity": 0,
  "ruleName": "Malware.Generic"
}

Regra:

{
  "target": "normalized.severity_id",
  "source": "severity",
  "type_cast": "to_int",
  "required": true
}

Execução:

  1. Source: severity → 0
  2. Default: 0 não é null, pular
  3. Pre-cast: nenhum
  4. Value-map: nenhum
  5. Type-cast: to_int(0) → 0
  6. Escrever: normalized.severity_id = 0

Resultado: Sucesso.

Construtor de Array (v2 somente)​

Tipo de regra especial para construir arrays de observáveis OCSF de múltiplas sources.

Forma:

{
  "target": "normalized.observables",
  "kind": "array_builder",
  "items": [
    {
      "name": "src_ip",
      "type": "IP Address",
      "type_id": 2,
      "source": "remoteIp"
    },
    {
      "name": "email_to",
      "type": "Email Address",
      "type_id": 5,
      "source": "recipients",
      "explode": true
    }
  ],
  "skip_null": true,
  "dedup_by": ["value"]
}

Cada item produz um dicionário observável no array de saída.

Campos de item:

CampoTipoObrigatório?
namestringsim
typestringsim
type_idintsim
sourceJMESPathsim
explodebooleannão
skip_nullbooleannão

Campos de nível de regra:

CampoTipoPadrão
skip_nullbooleantrue
dedup_bylist[string][]

Proibido no nível de regra: source, const, value_map, default, type_cast, pre_cast, fallback_source, when, required.

explode (nível de item): Se verdadeiro e a source resolver para uma lista, um observável é produzido por elemento. Se a source for escalar, envolvida em uma lista de um elemento.

skip_null (item & nível de regra): Se verdadeiro, valores null são omitidos. O nível de item sobrescreve o nível de regra.

dedup_by: Nomes de campos para deduplicar (primeira ocorrência vence).

Referência cruzada: Construtor de Array em Cookbook, Casos de Uso.

Tratamento de Erros​

Tempo de compilação (erros de definição de mapeamento)​

Lançados quando o mapeamento é analisado:

  • Sintaxe JMESPath inválida
  • Nome de type_cast ou pre_cast desconhecido
  • source e const ambos presentes
  • Campo obrigatório ausente
  • fallback_source usa raiz incompatível (raw vs extracted)

Resposta: MappingDefinitionError nos logs + HTTP 400 para UI.

Tempo de execução (erros de processamento de evento)​

Lançados quando aplicando regra a um evento:

  • type_cast falha (ex: string não numérica para to_int)
  • Campo obrigatório resolve para null
  • Fonte de item do construtor de array é inválida

Resposta: Evento quarentenado com error_kind: "map".
