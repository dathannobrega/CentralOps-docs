Referência de Operadores

Catálogo de todos os operadores disponíveis no DSL de normalização.

Type Casts​

Type casts transformam um valor após value_map ser aplicado. Aplicados em ordem de pipeline: default → pre_cast → value_map → type_cast.

iso_to_epoch​

Converte: String ISO-8601 (com ou sem Z) para segundos Unix epoch (int).

Tipo de entrada: str | int | float
Tipo de saída: int

Comportamento:

  • String "2026-04-27T12:30:00Z" → 1745764200 (exemplo de epoch).
  • String "2026-04-27T12:30:00+05:00" → ajusta corretamente para offset de timezone.
  • Int ou float (já é epoch) → passthrough inalterado (idempotente).
  • None → lança OperatorError (use default: null antes do cast para tratar ausente).

Exemplo:

{
  "target": "normalized.time",
  "source": "eventTime",
  "type_cast": "iso_to_epoch",
  "required": true
}

Atenção: Vendor envia "2026-04-27T12:30:00" (sem timezone)? Assumido UTC. Se seu vendor é um timezone diferente, você deve tratar offset no pré-processamento ou via camada de transformação.

epoch_to_iso​

Converte: Segundos Unix epoch (int ou float) para string ISO-8601 com sufixo Z.

Tipo de entrada: int | float | str
Tipo de saída: str

Comportamento:

  • Int/float 1745764200 → "2026-04-27T12:30:00Z".
  • String "1745764200" → analisada como float, depois convertida.
  • None → lança OperatorError.

Exemplo:

{
  "target": "normalized.last_seen_time",
  "source": "lastSeenEpoch",
  "type_cast": "epoch_to_iso"
}

Atenção: Se vendor envia epoch como string que falha ao analisar (ex: "not-a-number"), cast lança OperatorError e evento é quarentenado.

to_str​

Converte: Qualquer valor não-null para string.

Tipo de entrada: any (non-None)
Tipo de saída: str

Comportamento:

  • Int 42 → "42".
  • List [1, 2, 3] → "[1, 2, 3]" (representação estilo JSON).
  • Dict → representação JSON.
  • None → lança OperatorError.

Exemplo:

{
  "target": "normalized.severity",
  "source": "severity",
  "type_cast": "to_str"
}

Atenção: Use pre_cast: "to_str" ANTES de value_map se você está convertendo números para strings para busca. to_str após value_map geralmente é para formatar saída.

to_int​

Converte: String ou float para int.

Tipo de entrada: str | int | float
Tipo de saída: int

Comportamento:

  • String "42" → 42.
  • Float 42.7 → 42 (trunca, não arredonda).
  • Int 42 → 42 (passthrough).
  • Bool → lança OperatorError (bool é tecnicamente um int em Python; rejeição explícita previne bugs).
  • String "not-a-number" → OperatorError.

Exemplo:

{
  "target": "normalized.severity_id",
  "source": "severity",
  "type_cast": "to_int"
}

Atenção: Passar bool (de type_cast: "to_bool") para to_int falhará. Converta explicitamente se necessário: value_map: { "true": 1, "false": 0 }.

to_bool​

Converte: String, int, float para boolean.

Tipo de entrada: str | int | float | bool
Tipo de saída: bool

Comportamento:

  • String "true", "1", "yes", "y" (case-insensitive) → True.
  • String "false", "0", "no", "n", "" (vazio) → False.
  • Int/float: 0 ou 0.0 → False, qualquer coisa else → True.
  • Bool → passthrough.
  • String "maybe" → lança OperatorError (ambíguo).

Exemplo:

{
  "target": "normalized.is_terminal",
  "source": "isEndpoint",
  "type_cast": "to_bool"
}

Atenção: String "0" é truthy em Python (string não-vazia) mas to_bool a trata como False para padrões comuns.

score_to_percent​

Converte: Float [0.0, 1.0] para int [0, 100], ou valida int já em [0, 100].

Tipo de entrada: float[0..1] | int[0..100] | None
Tipo de saída: int[0..100] | None

Comportamento:

  • Float 0.75 → 75 (multiplica por 100, arredonda).
  • Float 0.755 → 76 (arredondamento do banqueiro, arredonda para par mais próximo).
  • Int 75 (já no intervalo) → 75 (passthrough).
  • Int 150 → OperatorError (fora do intervalo).
  • None → None (passthrough, sem erro).

Exemplo:

{
  "target": "normalized.confidence",
  "source": "alertScore",
  "type_cast": "score_to_percent"
}

Atenção: Se vendor envia score: 85 (já 0-100), este cast o rejeitará como fora do intervalo para float [0.0, 1.0]. Verifique o intervalo do vendor primeiro; se é 0-100 int, ignore este cast.

lowercase​

Converte: String para minúscula.

Tipo de entrada: str | None
Tipo de saída: str | None

Comportamento:

  • String "HTTP" → "http".
  • None → None (passthrough).
  • Não-string (int, bool) → OperatorError.

Exemplo:

{
  "target": "normalized.protocol_name",
  "source": "protocol",
  "type_cast": "lowercase"
}

Atenção: Caracteres Unicode são minúsculos por locale. Não garantido corresponder comparações case-insensitive exatas em todos os idiomas.

uppercase​

Converte: String para maiúscula.

Tipo de entrada: str | None
Tipo de saída: str | None

Comportamento:

  • String "http" → "HTTP".
  • None → None (passthrough).
  • Não-string → OperatorError.

Exemplo:

{
  "target": "normalized.tactic_name",
  "source": "mitreTactic",
  "type_cast": "uppercase"
}

trim​

Converte: String com espaço em branco removido de ambas extremidades.

Tipo de entrada: str | None
Tipo de saída: str | None

Comportamento:

  • String " example " → "example".
  • None → None (passthrough).
  • Não-string → OperatorError.

Exemplo:

{
  "target": "normalized.device.name",
  "source": "deviceName",
  "type_cast": "trim"
}

Atenção: Remove apenas espaço em branco à esquerda/direita, não espaços internos.

to_array​

Converte: Escalar para array de um elemento, ou passa através de arrays.

Tipo de entrada: T | list[T] | None
Tipo de saída: list[T] | list (vazio) | list (um elemento)

Comportamento:

  • Escalar "example" → ["example"].
  • List ["a", "b"] → ["a", "b"] (passthrough).
  • None → [] (lista vazia, não [None]).

Exemplo:

{
  "target": "normalized.finding_info.types",
  "source": "type",
  "type_cast": "to_array"
}

Atenção: to_array em None produz lista vazia, não [None]. Se você precisa [None], use const: [null] ou default: [] para evitar o cast de array.

dedup​

Converte: Lista com duplicatas removidas.

Tipo de entrada: list[T] | None
Tipo de saída: list[T] | None

Comportamento:

  • List [1, 2, 1, 3] → [1, 2, 3] (ordem da primeira ocorrência preservada).
  • Lista com itens não hash (dicts) → dicts são anexados mas não verificados para duplicatas (evita crashes).
  • Lista vazia [] → [] (passthrough).
  • None → None (passthrough).
  • Não-lista → OperatorError.

Exemplo:

{
  "target": "normalized.tags",
  "source": "tags",
  "type_cast": "dedup"
}

Atenção: Se sua lista contém dicts ou outros tipos não hash, dedup ainda funciona mas não realmente deduplica esses itens (volta a anexar). Para deduplicação complexa, use array_builder com dedup_by.

mitre_tactic_to_ocsf​

Converte: Forma de tática MITRE do Sophos para forma OCSF.

Tipo de entrada: list[{tactic: {id: str, name: str}}] | None
Tipo de saída: list[{tactics: [{uid: str, name: str}], version: "16.1"}] | None

Comportamento:

  • Forma do Sophos: [{"tactic": {"id": "TA0001", "name": "Initial Access"}}, ...]
  • Forma OCSF: [{"tactics": [{"uid": "TA0001", "name": "Initial Access"}], "version": "16.1"}, ...]
  • Táticas null (itens com tactic: null) são silenciosamente filtrados.
  • None → None (passthrough).
  • Itens malformados (faltando id ou name) → OperatorError.

Exemplo:

{
  "target": "normalized.attacks",
  "source": "mitreAttacks",
  "type_cast": "mitre_tactic_to_ocsf"
}

Atenção: Este cast é específico do Sophos. Se outro vendor envia táticas MITRE em uma forma diferente, você precisa de um cast diferente (ou uma feature v3 para mapeamento de forma).

Predicados​

Predicados são expressões booleanas usadas em portões when (v2 somente). Avaliados antes da execução da regra; se falso, regra é pulada.

exists​

Semântica: Verdadeiro se expressão JMESPath resolve para não-null.

Sintaxe:

{ "when": { "exists": "mailFrom" } }
{ "when": { "exists": "_processed.parsedAlert.fields.clientIp" } }

Comportamento:

  • Lista/dicionário vazio conta como exists (truthy no nível JSON).
  • None ou caminho ausente → não exists (falso).

Exemplo:

{
  "target": "normalized.actor.user.email_addr",
  "source": "mailFrom",
  "when": { "exists": "mailFrom" }
}

Atenção: Diferencie entre exists e equals. exists apenas verifica não-null; equals verifica um valor específico.

equals​

Semântica: Verdadeiro se expressão JMESPath iguala um valor literal (type-strict).

Sintaxe:

{ "when": { "equals": { "source": "direction", "value": "outbound" } } }

Comportamento:

  • Type-strict: int 3 != str "3".
  • Se source é null, predicado é falso (null nunca iguala nada).
  • Comparação é baseada em valor, não identidade.

Exemplo:

{
  "target": "normalized.traffic.direction",
  "source": "direction",
  "when": { "equals": { "source": "direction", "value": "outbound" } }
}

Atenção: Comparação de string case-sensitive. "Outbound" != "outbound".

in​

Semântica: Verdadeiro se expressão JMESPath é membro de uma lista de valores (type-strict).

Sintaxe:

{ "when": { "in": { "source": "severity", "values": ["critical", "high"] } } }

Comportamento:

  • Type-strict membership.
  • Se source é null, predicado é falso.
  • Ordem da lista de values não importa.

Exemplo:

{
  "target": "normalized.severity_id",
  "source": "severity",
  "value_map": { "critical": 5, "high": 4, "medium": 3 },
  "when": { "in": { "source": "severity", "values": ["critical", "high"] } }
}

Atenção: A lista values deve ser literal (não uma expressão JMESPath). Se você precisa membership dinâmica, use uma abordagem diferente.

not​

Semântica: Negação lógica de um predicado aninhado.

Sintaxe:

{ "when": { "not": { "exists": "errorField" } } }
{ "when": { "not": { "equals": { "source": "status", "value": "unknown" } } } }

Comportamento:

  • Recursivamente inverte o valor de verdade do predicado aninhado.
  • Se predicado aninhado é verdadeiro, not é falso; vice versa.

Exemplo:

{
  "target": "normalized.traffic.direction",
  "source": "direction",
  "when": { "not": { "equals": { "source": "direction", "value": "unknown" } } }
}

Atenção: Dupla negação é permitida mas confusa. Use com moderação.

Preprocess Ops​

Operadores de pré-processamento executam uma vez por evento antes de regras, populando o namespace _extracted (v2 somente).

json_parse​

Semântica: Analisa uma string JSON em um objeto Python.

Entrada: String-de-JSON (ou None).
Saída: Objeto Python analisado (ou None).

Sintaxe:

{
  "op": "json_parse",
  "source": "processedData",
  "target": "_processed",
  "tolerant": true
}

Campos:

  • source (obrigatório) — Expressão JMESPath para extrair a string JSON do evento bruto.
  • target (obrigatório) — Onde armazenar resultado analisado; DEVE começar com _.
  • tolerant (opcional, padrão: false) — Se verdadeiro, erros de análise silenciosamente retornam None; se falso, erros lançam e quarentenam evento.

Comportamento:

  • Entrada None → saída None (sem tentativa de análise).
  • Entrada String → objeto JSON analisado/array/string/número/bool.
  • Não-string (int, bool) → OperatorError (ou None se tolerant).
  • JSON malformado → OperatorError (ou None se tolerant).
  • Tamanho de string > PREPROCESS_JSON_MAX_BYTES → OperatorError SEMPRE (proteção DDoS; tolerant não suprime).

Exemplo:

{
  "preprocess": [
    {
      "op": "json_parse",
      "source": "processedData",
      "target": "_processed",
      "tolerant": true
    }
  ],
  "rules": [
    {
      "target": "normalized.src_endpoint.ip",
      "source": "_processed.parsedAlert.fields.clientIp"
    }
  ]
}

Ambiente: PREPROCESS_JSON_MAX_BYTES (padrão: 1048576, i.e., 1 MiB) controla tamanho máximo JSON. Não pode ser sobrescrito por regra; defina globalmente se vendor envia payloads grandes.

Atenção:

  • tolerant: true é recomendado para dados de vendor com problemas ocasionais de codificação.
  • Se string JSON contém sequências de escape (ex: \n), analisador as trata corretamente.
  • Defina target: "_extracted_email" ou similar para evitar colisões de namespace se múltiplos preprocess ops.

Tipos de Regra Especiais​

Regra Default (Escalar)​

Tipo de regra padrão (implícito em todos os mapeamentos v1 e maioria v2).

Produz: Valor escalar único ou null.

Exemplo:

{
  "target": "normalized.severity_id",
  "source": "severity",
  "type_cast": "to_int"
}

array_builder​

Tipo de regra especial para construir arrays de observáveis OCSF.

Campos obrigatórios:

  • target — ex: "normalized.observables"
  • kind: "array_builder"
  • items — lista de especificações de observável

Campos opcionais:

  • skip_null (padrão: true) — omitir itens com sources null
  • dedup_by (padrão: []) — campos para dedup

Campos de item:

  • name, type, type_id (obrigatório) — metadados de observável
  • source (obrigatório) — JMESPath para extrair valor
  • explode (padrão: false) — se verdadeiro e source é lista, um observável por elemento
  • skip_null (padrão: nível de regra) — sobrescrever skip_null de nível de regra para este item

Proibido no nível de regra: source, const, value_map, default, type_cast, pre_cast, fallback_source, when, required.

Proibido em itens: required — itens são passthrough; sem quarentena em null.

Exemplo:

{
  "target": "normalized.observables",
  "kind": "array_builder",
  "items": [
    {
      "name": "src_ip",
      "type": "IP Address",
      "type_id": 2,
      "source": "_processed.parsedAlert.fields.clientIp"
    },
    {
      "name": "email_to",
      "type": "Email Address",
      "type_id": 5,
      "source": "_processed.parsedAlert.fields.envelopeRecipients",
      "explode": true
    }
  ],
  "dedup_by": ["value"]
}
